## 🎯 面向对象分析成果

### 1. 识别的核心对象

```
User（用户）
├── Student（学生）
│   ├── has many Answer（答题记录）
│   ├── has many Mistake（错题记录）
│   └── has one Statistics（学习统计）
├── Teacher（教师）
└── Admin（管理员）

Problem（题目）
├── has many Answer
└── has many Mistake

TrainingSession（训练会话）
├── belongs to Student
└── has many Answer

Statistics（统计数据）
├── belongs to Student
└── has many DailyStatistic
```

### 2. 关键的业务规则

| 规则     | 描述                                   |
| -------- | -------------------------------------- |
| 等级晋升 | 最近 20 题正确率 ≥ 90% 且当前等级 < 10 |
| 错题记录 | 答题错误时自动记录                     |
| 账号锁定 | 密码连续错误 3 次时锁定 30 分钟        |
| 会话超时 | 15 分钟无操作需重新登录                |
| 评分规则 | 基础分 + 耗时奖励（答题越快加分越多）  |

### 3. 系统工作流

```
登录认证 → 首页展示 → 功能选择
              ├→ 开始训练 → 参数配置 → 生成题目 → 答题循环 → 结束会话
              ├→ 查看错题 → 筛选 → 管理（改正/删除）
              └→ 查看统计 → 多维度分析
```
