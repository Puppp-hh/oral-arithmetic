# 实验5：口算小程序用例分析

## 1. 系统概述

本实验以当前的"小学数学口算分级训练小程序"为对象，采用面向对象分析方法，围绕系统中的参与者和主要业务目标进行用例建模。

系统主要支持以下核心业务：

- 用户认证与身份管理（学生注册、学生/教师登录、退出登录）
- 自适应题目训练（难度与题型选择、题目生成、自动判题）
- 错题本管理（查看、筛选、标记改正、删除）
- 学习数据分析（实时统计、历史数据、题型分布）
- 进度跟踪（等级晋升、正确率统计）

## 2. 系统边界与参与者

### 2.1 系统边界

**系统名称**：口算小程序系统

**系统范围**：

- 前端：微信小程序客户端（学生端和教师端）
- 后端：Express.js 服务器 + MySQL 数据库 + Redis 缓存
- 包含的功能：用户认证、题目训练、答案评判、错题管理、学习统计

**系统外的对象**：

- 微信平台（登录授权）
- 系统管理员（数据初始化，本用例图不涉及）

### 2.2 参与者（Actor）

#### 2.2.1 学生（Student）

- **定义**：系统的主要使用者，持有学生身份的合法账号
- **职责**：
  - 注册和登录系统
  - 选择难度和题型进行训练
  - 查看和管理错题本
  - 查看个人学习统计
- **特征**：可进行训练、查看错题、查看统计等大部分系统操作

#### 2.2.2 教师（Teacher）

- **定义**：持有教师身份账号，在系统中有有限权限
- **职责**：
  - 使用教师账号登录系统
  - 查看系统首页（展示学校数据概览）
- **特征**：权限受限，主要用于数据查看，不涉及训练和错题管理

## 4. 用例文档

### 4.1 认证模块用例

#### UC-00：执行登录认证

| 属性         | 内容                                                    |
| ------------ | ------------------------------------------------------- |
| **用例编号** | UC-00                                                   |
| **用例名称** | 执行登录认证                                            |
| **类型**     | 子用例（被包含）                                        |
| **参与者**   | 学生、教师                                              |
| **目标**     | 系统对用户提供的账号密码进行验证并生成认证令牌          |
| **前置条件** | 用户已输入账号和密码                                    |
| **后置条件** | 系统返回有效的 JWT Token 和用户信息，或返回认证失败消息 |

**基本流程**：

1. 后端收到登录请求，包含账号、密码、身份类型（学生/教师）
2. 系统验证账号是否存在且身份类型相符
3. 系统通过 bcrypt 对比提供的密码和存储的密码哈希值
4. 验证成功后，系统生成 JWT Token（包含用户ID、身份、过期时间）
5. 系统将 Token 存入 Redis 并设置会话超时
6. 系统返回 Token 和用户基本信息（ID、账号、姓名、角色）

**备选流程**：

- **A1**：账号不存在
  - 系统返回错误消息 "账号不存在或身份类型错误"
  - 用户返回登录输入界面

- **A2**：密码错误（连续 3 次）
  - 系统记录登录失败次数
  - 达到 3 次后，系统锁定账号 30 分钟
  - 用户收到 "账号已被锁定，请稍后重试" 消息

- **A3**：数据库查询失败
  - 系统记录错误日志
  - 系统返回 "服务异常，请稍后重试"

#### UC-01：学生登录

| 属性         | 内容                                                              |
| ------------ | ----------------------------------------------------------------- |
| **用例编号** | UC-01                                                             |
| **用例名称** | 学生登录                                                          |
| **参与者**   | 学生                                                              |
| **目标**     | 学生使用账号和密码登录系统，进入首页                              |
| **前置条件** | 学生已拥有有效账号或需注册新账号                                  |
| **后置条件** | 系统将 Token 保存到本地，学生进入首页，可访问训练、错题、统计功能 |

**基本流程**：

1. 学生打开小程序，进入登录页面
2. 系统检查本地是否存在有效的 Token，若存在则自动跳过登录
3. 学生在登录页选择"学生"身份
4. 学生输入账号和密码
5. 学生点击"登录"按钮
6. 前端校验账号和密码的格式（账号长度 4-16 位，密码长度 6-20 位）
7. 前端向后端发送登录请求（身份类型=学生）
8. 系统执行 UC-00（执行登录认证）
9. 认证成功后，前端保存 Token 到本地存储和内存
10. 前端发起请求获取学生的学习摘要（等级、正确率等）
11. 系统跳转至首页

**备选流程**：

- **A1**：学生选择了"教师"身份
  - 系统提示 "请确认身份类型"
  - 学生可切换回学生身份

- **A2**：账号或密码为空
  - 系统弹出提示 "账号和密码不能为空"
  - 学生返回输入框继续输入

- **A3**：账号或密码格式错误
  - 系统弹出提示 "账号或密码格式不正确"
  - 学生可重新输入或选择注册

- **A4**：认证失败（来自 UC-00 的异常流程）
  - 系统显示对应的错误提示
  - 学生点击"重新登录"返回输入界面

- **A5**：网络连接异常
  - 系统提示 "网络连接失败，请检查网络后重试"
  - 学生可点击"重试"或"离线浏览"

#### UC-03：学生注册

| 属性         | 内容                                           |
| ------------ | ---------------------------------------------- |
| **用例编号** | UC-03                                          |
| **用例名称** | 学生注册                                       |
| **参与者**   | 学生                                           |
| **目标**     | 新学生创建账号，初始化个人档案                 |
| **前置条件** | 学生首次打开应用或选择注册                     |
| **后置条件** | 系统创建新的学生账号，自动登录，初始化学生档案 |

**基本流程**：

1. 学生在登录页点击"注册"按钮
2. 系统跳转至注册页
3. 学生输入学生账号（4-16 位）、设置密码（6-20 位）、确认密码
4. 学生输入个人信息：姓名、年级、班级、性别
5. 学生点击"注册"按钮
6. 前端校验输入的有效性
7. 前端向后端发送注册请求
8. 后端检查账号是否已存在
9. 若账号未被占用，后端使用 bcrypt 加密密码
10. 后端创建新的学生记录，初始化数据（等级=1，正确率=0%，训练题数=0）
11. 后端自动为学生生成并返回 Token
12. 前端保存 Token 并跳转至首页

**备选流程**：

- **A1**：账号已被占用
  - 系统提示 "账号已存在，请重新输入或直接登录"
  - 学生返回选择新账号

- **A2**：两次输入的密码不一致
  - 系统提示 "两次密码输入不一致"
  - 学生重新输入

- **A3**：姓名、班级等必填项为空
  - 系统提示 "请完整填写所有必填项"
  - 学生继续填写

- **A4**：注册时数据库服务异常
  - 系统提示 "注册失败，请稍后重试"

#### UC-10：退出登录

| 属性         | 内容                                      |
| ------------ | ----------------------------------------- |
| **用例编号** | UC-10                                     |
| **用例名称** | 退出登录                                  |
| **参与者**   | 学生、教师                                |
| **目标**     | 用户主动结束当前会话，清除本地认证信息    |
| **前置条件** | 用户已登录                                |
| **后置条件** | 系统清除本地 Token 和用户信息，返回登录页 |

**基本流程**：

1. 用户点击首页或菜单中的"退出登录"选项
2. 系统弹出确认对话框："确定要退出登录吗？"
3. 用户点击"确定"
4. 前端向后端发送登出请求，携带当前 Token
5. 后端从 Redis 中删除该 Token 的会话信息
6. 后端返回成功响应
7. 前端清除本地存储中的 Token、用户信息、登录状态
8. 前端清除所有页面缓存数据
9. 系统跳转至登录页
10. 用户可重新登录或选择其他操作

**备选流程**：

- **A1**：用户点击"取消"
  - 系统关闭对话框，停留在当前页面

- **A2**：登出请求失败
  - 前端仍然清除本地数据（确保本地和服务器最终一致）
  - 系统显示 "登出异常，已清除本地数据"
  - 系统跳转至登录页

### 4.2 首页模块用例

#### UC-04：查看首页概览

| 属性         | 内容                                               |
| ------------ | -------------------------------------------------- |
| **用例编号** | UC-04                                              |
| **用例名称** | 查看首页概览                                       |
| **参与者**   | 学生、教师                                         |
| **目标**     | 用户进入首页后查看个人信息、等级进度和功能快捷入口 |
| **前置条件** | 用户已登录且 Token 有效                            |
| **后置条件** | 系统展示个性化的首页信息                           |

**基本流程**（学生视角）：

1. 用户登录后自动进入首页或用户从其他页面点击"首页"标签
2. 系统从本地取出 Token，验证 Token 有效性
3. 系统向后端发起请求，获取学生的学习摘要
4. 后端查询该学生的最新档案数据（等级、正确率、今日答题数等）
5. 系统展示学生信息卡片：
   - 学生姓名、年级、班级
   - 当前等级（1-10 级）
   - 累计正确率（百分比）
   - 今日已答题数
6. 系统展示等级进度条，显示当前等级的完成度
7. 系统展示快捷功能入口：开始训练、查看错题本、学习统计、设置、关于
8. 页面加载完成

**基本流程**（教师视角）：

1. 教师登录后进入首页
2. 系统验证 Token 有效性
3. 系统向后端请求教师所属学校的统计数据（学生总数、平均正确率、最近 7 天活跃学生数）
4. 后端返回学校级别的聚合统计数据
5. 系统展示教师首页：
   - 欢迎信息："欢迎，[教师名]"
   - 学校统计信息卡片
   - 快捷功能入口（仅限：查看学校数据、关于、退出登录）
6. 页面加载完成

**备选流程**：

- **A1**：本地 Token 已过期或无效
  - 系统清除本地 Token
  - 系统跳转至登录页
  - 用户需重新登录

- **A2**：后端返回用户不存在
  - 系统清除本地登录状态
  - 系统提示 "账户异常，请重新登录"
  - 系统跳转至登录页

- **A3**：加载学习摘要失败（网络异常或服务异常）
  - 系统显示加载中动画，自动重试 3 次
  - 若仍失败，系统展示骨架屏内容，提示 "数据加载异常，请下拉刷新"
  - 用户可手动下拉刷新重新加载

- **A4**：部分数据加载失败（如等级进度）
  - 系统展示已加载的内容（用户卡片）
  - 未加载的部分显示 "加载中..."
  - 系统继续后台加载

**关键需求**：

- 首页应在 2 秒内完成加载
- 如果用户长时间未操作（15 分钟），系统应自动刷新 Token，如刷新失败则要求重新登录
- 首页数据在每次进入时都应从服务器更新（不使用过期的本地缓存）

### 4.3 训练模块用例

#### UC-05：开始训练会话

| 属性         | 内容                                               |
| ------------ | -------------------------------------------------- |
| **用例编号** | UC-05                                              |
| **用例名称** | 开始训练会话                                       |
| **参与者**   | 学生                                               |
| **目标**     | 学生选择训练参数（难度、题型、数量），启动训练流程 |
| **前置条件** | 学生已登录                                         |
| **后置条件** | 系统生成题目列表，进入答题界面                     |

**基本流程**：

1. 学生从首页点击"开始训练"或从菜单进入训练页
2. 系统加载训练参数配置页
3. 系统展示参数配置表单，包含以下选项：
   - **难度等级**（单选，1-10 级）：默认为当前学生的推荐等级
   - **题型**（多选）：加法、减法、乘法、除法、混合运算
   - **题目数量**（单选）：10 题、20 题、30 题，默认 20 题
4. 学生根据需要调整参数并点击"开始训练"按钮
5. 系统校验参数合法性
6. 系统向后端发送题目生成请求
7. 后端生成题目列表（最多 30 题）
8. 系统初始化训练状态（当前题号=1，已答题数=0，正确数=0）
9. 系统展示第一道题目，启动计时
10. 学生开始答题

**备选流程**：

- **A1**：生成题目时后端返回空列表
  - 系统提示 "未能生成题目，请检查参数设置或稍后重试"
  - 学生返回参数配置页重新选择

- **A2**：用户在训练中点击"退出训练"
  - 系统弹出确认对话框："是否放弃本次训练？当前进度将丢失"
  - 若用户确认，系统清空训练状态，返回首页
  - 此次训练的答题记录不会被保存

- **A3**：网络中断
  - 系统将当前答题状态缓存到本地
  - 系统提示 "网络连接断开，请恢复网络后继续"
  - 用户恢复网络后，系统自动恢复训练状态

| **参与者**   | 学生、系统                           |
| **目标**     | 学生提交答案，系统自动判题并记录结果 |
| **前置条件** | 学生已输入答案                       |
| **后置条件** | 系统记录答题结果，更新统计数据       |

**基本流程**：

1. 学生点击"提交"按钮
2. 系统校验答题框不能为空，若为空则提示 "请输入答案"
3. 学生确认答案后，系统停止计时，记录答题耗时
4. 系统向后端发送答题请求，包含：题目ID、学生答案、答题耗时、难度等级
5. 后端进行判题：
   - 若完全相同，判定为"正确"
   - 若答案为空或明显不符，判定为"错误"
6. 后端计算评分：
   - 基础分 = 正确时 100 分，错误时 0 分
   - 耗时调整：答题时间越短，加分越多
   - 最终得分 = max(0, 基础分 + 耗时奖励)
7. 后端向数据库保存答题记录
8. 若答题错误，后端向数据库保存错题记录
9. 系统展示判题结果：
   - 您的答案：[答案]
   - 标准答案：[答案]
   - 判题结果：✓ 正确 / ✗ 错误
   - 获得分数：XX 分
10. 学生查看结果后，点击"下一题"继续

**备选流程**：

- **A1**：答题请求失败（网络异常）
  - 系统重试 3 次
  - 若仍失败，系统提示 "提交失败，请稍后重试"

- **A2**：后端返回题目不存在
  - 系统提示 "题目异常，训练无法继续"
  - 学生返回首页

#### UC-08：结束训练会话

| 属性         | 内容                                     |
| ------------ | ---------------------------------------- |
| **用例编号** | UC-08                                    |
| **用例名称** | 结束训练会话                             |
| **参与者**   | 学生                                     |
| **目标**     | 系统计算训练总结，更新学生等级和统计数据 |
| **前置条件** | 学生已答完所有题目或主动退出训练         |
| **后置条件** | 系统生成训练报告，更新学生档案           |

**基本流程**：

1. 学生答完最后一题或点击"结束训练"
2. 系统向后端发送训练结束请求
3. 后端计算本次训练的统计数据：
   - 总题数、正确题数、错误题数
   - 本次正确率、总得分
4. 后端更新学生档案：
   - 累计训练题数 += N
   - 累计正确题数 += C
   - 累计正确率重新计算
5. 后端检查是否满足等级晋升条件：
   - 条件：最近 20 题正确率 >= 90% 且当前等级 < 10
   - 若满足，晋升等级
6. 后端返回训练总结信息
7. 系统展示本次训练总结页：
   - 本次成绩（题数、正确数、正确率）
   - 等级进度
   - 错题预览
8. 用户点击"返回首页"或"继续训练"

**备选流程**：

- **A1**：训练结束请求失败
  - 系统缓存训练数据到本地
  - 系统提示 "结果保存失败，将在网络恢复时上传"
  - 用户下次登录时，系统自动上传缓存数据
